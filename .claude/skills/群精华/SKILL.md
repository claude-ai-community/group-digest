# 工作流:群精华

**触发词:**「群精华」「整理群聊」「群精华 raw/xxx.md」
**输入:** 二选一 —— ① 用户粘贴聊天记录(原始消息或小微总结);② `raw/` 下的文件(用户先把小微总结存进去、编辑补充后,指定文件名)
**产出:** `digests/YYYY-MM-DD.md` + 更新 `tools.md` / `tips.md` / `README.md` 索引 + git commit push

---

## Step 1:收输入

两种来源:
- **粘贴**:用户直接贴聊天内容。
- **raw 文件**:用户说「群精华 raw/2026-07-06.md」→ 读该文件。只说「群精华」没贴内容也没指定文件时,列出 `raw/` 下未处理的文件(对照 digests/ 已有日期)让用户挑。

日期:raw 文件名里的日期优先;否则用今天;聊天跨多天问一句。

> raw/ 已 gitignore(公开库,原始材料含真名,永不上传)。处理完的 raw 文件保留在本地作底稿,不删。

判断输入类型:
- **原始聊天记录**(带昵称/时间戳)→ 走完整提炼。
- **小微总结**(已是概括文本)→ 直接结构化,但提醒用户:小微可能丢了链接和细节,重要讨论建议补原文。

## Step 2:匿名化(强制,先做)

- 微信昵称 → 拼音/英文首字母(如「张三」→ Z 同学、「Mike Wang」→ M)。同一人全文保持同一代号。
- 删除:微信号、手机号、邮箱、二维码描述、任何联系方式。
- Austin 本人发言可署名 Austin。

## Step 3:提炼(不搬运)

从聊天里抽四类东西,**按话题归纳,不逐条搬运**:

1. **工具** — 工具名、谁用过、一句话评价、链接(原文有链接必须保留原样)
2. **心得/经验** — 场景 + 做法 + 效果/坑
3. **讨论主题** — 有来回讨论的话题,总结正反观点和结论
4. **金句/亮点**(可选) — 特别精彩的原话(匿名)

判断力:闲聊、寒暄、纯表情、跑题内容直接丢弃。宁缺勿滥。

## Step 4:写精华文件

写 `digests/YYYY-MM-DD.md`:

```markdown
# 群精华 · YYYY-MM-DD

> 来源:AI 飞轮实践群 · 整理:Austin + AI · 群友已匿名

## 本期看点
<一两句概括本期最值得看的>

## 工具
<每个工具:名字加粗 + 一句话 + 链接 + 谁推荐(代号)>

## 心得
<场景 + 做法 + 效果>

## 讨论
<话题 + 观点归纳 + 结论(如有)>

## 金句
<可选>
```

## Step 5:回填累积库

- **tools.md**:新工具加进对应类别(查重,已有的不重复;有新评价可补充)。收录日期填今天。
- **tips.md**:新心得按模板追加,来源指向本期 digest。
- **README.md**:精华索引表加一行(日期 + 主题 + 链接),最新在上;首期时删掉「(待首期)」占位行。

## Step 6:确认 + 发布

1. 给用户看一眼 digest 概要(标题 + 各节条数),问「发布吗?」。
2. 用户确认后:
```bash
git add -A && git commit -m "digest: YYYY-MM-DD 群精华" && git push
```
3. 回复用户 GitHub 链接,方便转发回群。

## 红线

- 匿名化是强制步骤,发布前自查一遍没有真名/联系方式。
- 只沉淀知识,不沉淀八卦/私事/敏感话题。
- 拿不准该不该收录的内容,问 Austin,别自作主张发布。
