## 本期看点

Austin 用 Harness 框架重新解构了乔哈里视窗——认知边界的移动不再只靠"更好的 prompt",而是四种可以工程化设计的新机制。同一条思路在 T 的 Prompt 设计心得里再次印证:只给验收标准和边界,别给 how。另外 MU 分享的 Claude Code 架构拆解讲透了"agentic search 为什么打赢 RAG",求是 AI 圆桌的工业/医疗闭门交流也带来了不少一线判断。

## 工具

- **Markitdown skill**(Claude 内置,微软提供) — 把 word/PDF 先转成 markdown 再放进 project 里,比每次用 Claude 内置读取器直接读原始文件更省 token。来源:Austin

## 心得

- **场景**:想省 token 又要处理大量 word/PDF 文档。
  **做法**:先用 Markitdown skill 把文档转成 markdown,一次转换后放进 project 里复用,而不是每次都用 Claude 内置工具读原始文件。
  **效果 / 坑**:更省 token;但 MU 提醒——先别急着优化 token,先图省事省力,等真的 scale up 了再考虑效率。
  贡献:Austin(方案,[相关视频](https://www.youtube.com/watch?v=1gYuZk-i95g))+ MU(优先级提醒)

- **场景**:怎么写出好的 Prompt / 验收标准,而不是手把手教模型解题步骤。
  **做法**:① 不写解题步骤,只写清楚"什么叫解完" ② 把定义、范围和边界情况一次性说清楚 ③ 不只告诉模型什么算答案,还要列出什么不算答案 ④ 复杂任务不固定分工,而是动态搜索,并设置独立审查。
  **效果 / 坑**:Austin 点评这正是"只给验收标准和边界,不给 how"(即 /goal 思路);并预判未来 agent 会变薄,像 langgraph 这类 graph flow 编排会被大模型自己端到端接管。
  贡献:T(四点心得,[原文](https://mp.weixin.qq.com/s/GhqoYu2DFDra5i51m1TIHQ))+ Austin(点评延伸)

## 讨论

- **乔哈里视窗的局限,Harness 框架怎么补**(本期最硬核的一段,Austin 原创分析,[原文](https://mp.weixin.qq.com/s/3GZlErBnfD2_pYu4a33UHQ)):原文用乔哈里视窗分析人机认知边界,框架清晰,但有一个根本局限——把"边界移动"锁死在 prompt 上,是静态窗框模型(只有"更好的 prompt"和"更大接收容量"两条路)。Austin 提出 Harness 框架下,边界移动其实有四种不依赖 prompt 的新机制:① **Skill 内化**(MCE/SkillOpt)——高价值轨迹被压缩成可复用 skill 文件,隐藏区被系统永久压缩,agent 下次执行不需要人重新解释;② **Sensor 设计**(Böckeler 的 guides + sensors)——harness 系统化捕获测试通过率、benchmark 分数、执行轨迹等人没说出口的信号,是系统观察代替人暴露;③ **主动上下文索取**——进化成熟的 agent 会主动问"你的意图是什么",隐藏区缩小由 agent 发起而非人发起;④ **Harness 代码自我修改**(DGM/AlphaEvolve)——agent 读自己的 benchmark 日志、发现执行漏洞、直接修改 harness 代码,窗框本身被重新设计。关键翻转:原文的盲目区是静态能力,Harness 框架下盲目区是动态扩张的——agent 能不断扩展自己"发现能力"本身。核心结论:**边界是工程对象,不只是沟通对象,设计比表达更根本**。GK 补充:这套机制成立的前提是 agentic 框架本身要透明可控,这也是他做自己"龙虾和河马"项目的出发点。

- **Claude Code Deep Dive:agentic search 为什么打赢 RAG**(MU 分享,[原文](https://claude.ai/public/artifacts/af01b1e3-c872-4103-8273-00271f82e5f8)):Claude Code 的强大来自三层协同——一个极简的 agentic loop("gather context → take action → verify work → repeat")、一套刻意 Unix 化的原语工具(bash/Read/Edit/Grep/Glob)、以及一个专门针对可验证软件工程任务做过 RL 训练的前沿模型。三者哲学一致:把智能留在模型里,把环境交还给模型自己探索,符合"the Bitter Lesson"。最反直觉的设计是"agentic search 取代 RAG/embedding 索引"——Claude Code 不预先索引代码库,而是像人类工程师一样用 grep/glob/read 按需实时探索。创建者 Boris Cherny 在 Latent Space 播客证实,早期版本其实用 Voyage 做过索引,但最终发现纯 agentic search"outperformed everything, by a lot",解决了索引陈旧、精度、零配置和隐私四个问题,代价是运行时更慢、token 消耗有"最坏情况爆炸"风险。给资深工程师的启示:agent 的竞争力更多来自 harness(权限管线、compaction 算法、工具设计、context 工程)而非单纯模型,但模型侧的 long-horizon agency(Sonnet 4.5 可专注 30+ 小时)和 RLVR 式训练是 harness 能做到极简的前提;构建自己的 agent 时应"do the simplest thing that works",用少而正交的高杠杆工具 + 明确的 context 预算管理,而非堆砌工具或复杂 RAG 基础设施。Austin 评价"好文"。

- **求是 AI 圆桌:工业 / 医疗闭门交流核心观点**([工业](https://mp.weixin.qq.com/s/pmvwfINfMvOe7qax2VTCvw) / [医疗](https://mp.weixin.qq.com/s/yzFk9aeP-I7PVr34GnRnNg)):
  - **AI + 工业(7 月 4 日)**:AI 在工业领域的价值不能停留在通用模型展示,必须进入真实产线、流程和业务系统。陈高提出"AI 的边界是个人认知的边界,要以输出倒逼输入",举农药有效期管理场景为例,准确率从 76% 提升到 92%;徐诚雄强调工业 AI 必须真正进入产线,从需求挖掘到规模化落地需要实操路径;王东霞从多智能体系统、智能工控系统、可信 AI 角度探讨深度应用。圆桌基调是"所有人都是主角"的深度交流,而非单向讲座。
  - **AI + 医疗(7 月 2 日)**:同样不能停留在通用模型或单点算法展示,必须进入真实临床、科研与产业流程。医疗行业高度保守、强监管、强场景依赖,难点在可信数据、理解医生工作流、实验/临床验证闭环、可持续商业模式;场景选择不应追求大而全,应聚焦垂直场景(护理、手术、呼吸慢病、细胞培养、药物研发、数字病理等);医院数据涉及隐私合规,可信数据的获取治理能力是关键门槛;必须通过湿实验、动物实验或真实世界临床流程验证。代表性实践:Regelife 马昊分享的 AI 驱动材料配方设计引擎,把 18 个月试错压缩到秒级/分钟级,用双 Agent 方式控制幻觉风险;中科院杭州医学所李晓林建立"原子级操作系统"与生命科学 AI 平台,贯通文献阅读-数据分析-分子设计-实验验证闭环,个性化 mRNA 肿瘤疫苗基于患者个体数据完成新抗原识别与序列优化。
  - 整体判断:AI 与医疗/工业的结合已跨越概念验证阶段,真正可落地的项目往往围绕明确场景、可信数据闭环和可验证工作流展开。
