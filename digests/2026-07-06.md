## 工具

> 以下多数工具出自 Kun Chen 的工作流分享(Austin 转发),不是群友本人长期使用后的评测——收录供参考,标注了来源。J 正计划把这套方法和自己惯用的 opsx-superpowers 结合试用。

- **WezTerm / tmux / Neovim** — 终端驱动的 Agent 工程"船只"三件套:高性能可定制终端 + 窗格切分/跨设备保会话 + 高效编辑,维持不离键盘的深度心流。来源:Kun Chen(经 Austin 转发)
- **Open Super Whisper** — 本地运行的语音转录工具,语音输入比打字快 3 倍,仅路径/URL 这类内容才切回键盘。来源:Kun Chen(经 Austin 转发)
- **Lavish**(遵循 **AXI** 设计原则) — Kun Chen 自己开发的可视化规划器,解决 AI 规划阶段"文字墙"的问题:自动生成 HTML 原型,可在浏览器里直观审阅、标注、反馈。AXI 是他提出的"AI 优化工具设计标准",比传统 JSON/MCP 更省 token、延迟更低。来源:Kun Chen(经 Austin 转发)
- **No Mistakes 流水线** — 自动化 PR 处理:拉分支 → 对抗性审核 → 跑端到端测试 → 更新文档 → 修 lint,只有被标记高风险的改动才需要人介入。来源:Kun Chen(经 Austin 转发)
- **GNHF(Good Night, Have Fun)** — 处理长达 8 小时的长任务(自动找 bug、持续调优),可设 token 上限和停止条件,适合睡前挂机跑。来源:Kun Chen(经 Austin 转发)
- **Treehouse** — 自动管理 git worktree,让多个智能体在独立目录并行工作、互不干扰。来源:Kun Chen(经 Austin 转发)
- **First Mate(大副)** — 最高级别自动化:接受跨项目的模糊指令,自动调度多个智能体、建工作区、跑流水线、交付 PR,让"船长"专注战略。来源:Kun Chen(经 Austin 转发)
- **OpenSquilla**([opensquilla.ai](https://www.opensquilla.ai/zh/invite/#how)) — 开源多模型协同 Agent,5.4k star。按任务难度自动路由便宜/强模型,不用人肉切换;公开测评显示多模型分工协同、一个模型汇总,效果不输更贵方案、成本更低。**进群送 $10 API 额度**。分享:Austin

## 心得

- **场景**:claude.md 文件越写越大,占空间又难维护。
  **做法**:把里面"有条件才生效"的 section 交给 Claude Code 自动抽取,拆成独立 skill,实现"渐进式披露"。
  **效果 / 坑**:显著瘦身、条件逻辑模块化;⚠️ 从网上随便下载未经验证的 skill 有安全和性能风险,别乱装。
  贡献:G(方法)、Kun Chen 原文补充警示

- **场景**:多个 AI 工具(Claude Code、其他 agent)各管一份记忆文件,项目背景要重复维护。
  **做法**:用 symbolic link 让不同 AI 工具共享同一份记忆文件(分全局个人偏好 + 项目级代码规范/架构说明两层)。
  **效果 / 坑**:一处更新、处处生效,省去重复维护。
  来源:Kun Chen(经 Austin 转发)

- **场景**:工程师亲自审阅每一个 AI 生成的 diff,累。
  **做法**:心态从"写代码的人"转成"工程主管(Director)"——配合 No Mistakes 流水线,只在系统标记高风险时才深度介入。
  **效果 / 坑**:是"船长-船员"模式能规模化的关键心态转变。
  来源:Kun Chen(经 Austin 转发)

## 讨论

- **"船长-船员"完整工作流**:Kun Chen 分五层构建 —— ① 终端为中心的"船只"基础设施(WezTerm/tmux/Neovim)② 记忆与技能训练"船员"(symlink 共享记忆 + skill 渐进式披露,警惕未经验证的第三方 skill)③ 语音 + 可视化协作(Open Super Whisper、AXI 原则、Lavish)④ 质量把控(Director 心态 + No Mistakes 流水线,只审高风险改动)⑤ 规模化并行(GNHF 挂机长任务、Treehouse 管 worktree、First Mate 顶层调度)。结果:每天稳定交付 40-50 个生产级 commit。J 已在看原视频、计划结合自己的 opsx-superpowers 试用。
- **claude.md 泄露事件**:Andrej Karpathy 的 claude.md 疑似流出(见某 skills 仓库),内容新增了六条,但 J 指出**真实性存疑**,群里没有定论。
- **Lavish vs Claude Design**:辨清两者不是竞品——Lavish 做的是"AI 规划阶段可视化"(生成 HTML 原型给人审阅),Claude Design/Artifacts 是做实际设计产出,场景不同,Lavish 更轻量、能直接在 visual layout 上评论。
