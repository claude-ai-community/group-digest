## 讨论

### AI 编程与 Spec 设计

- Austin 分享 SSD(Spec-Software Development)驱动开发实践,核心观点:spec 应该是"架构设计的极限压缩版"——一屏摘要 + 关键图 + 契约/不变量 + 验收,让人三分钟审得动。([原文](https://mp.weixin.qq.com/s/CkstxNencCPu0mAI3D1vIQ))
- 提到 Martin Fowler 关于 DSL 和 AI 结合的文章。([原文](https://martinfowler.com/articles/llm-and-dsls.html))
- 讨论如何把 open spec 和 superpowers 结合起来,用 fable 做整合:Austin 用 superpowers 的 brainstorming 帮自己想怎么结合两者,并给了一个引入 harness 思路的实际 spec 案例。([案例](https://github.com/austinxyz/opsx-superpowers/blob/main/docs/superpowers/specs/2026-05-25-harness-validation-design.md))

### Agent 架构与模型对比

- G 提出 multi-agent 结构:fable 做 orchestrator,sonnet 做执行 workers,在别人代码基础上改出这个结构并开源。([GitHub](https://github.com/gwang-indoc/fable-the-boss))
- Austin 回应:multi-agent 不是每次都需要,但编程这类复杂工作可能需要;并抛出三个观察——
  - 看到一篇报道:Sonnet 5.0 虽然比 Opus 4.8 便宜,但因为成功率低、需要迭代次数多,最后单位成本反而更高
  - 追问:长任务丢给 Fable,它是不是会自动切分子任务,不需要的子任务也会自动跳过?
- **Sonnet 5 vs Opus 4.8 的真实成本**(Databricks 研究):仅按 token 单价比较会误导,应按"每完成一个任务的成本"衡量——Sonnet 5 单价更低,但完成率较低(81% vs 87%)且消耗更多 token,实际每任务成本反而更高。
- **Kimi K3 发布成为热点**:首个开源 3T 级模型(2.8 万亿参数),性能接近 Claude Fable 5,价格只要一半;代码任务成功率 92%,权威评测榜单上首次有开源模型超越全部海外闭源模型。([原文一](https://mp.weixin.qq.com/s/BWOeh0usqnMm-uSfFOUoNQ) / [原文二](https://mp.weixin.qq.com/s/J-ql5I9GxfZdeEp0H8s-bg))Bloomberg 称月之暗面发布 Kimi K3 是"第二次 DeepSeek 时刻",当天导致 AI 股票大跌,波及科技股。
- **本地部署模型的现实问题**:27B/35B 模型在消费级硬件上可用,但复杂任务仍需云模型——想装最新 GLM 5.2 可能要好几张 RTX 5090/6000 GPU 或 H100,几万美元;装 Qwen 3.6 的 27B 或 35B 模型,消费级 Mac Studio 或 NV 的 DGX 就够,大多数编程任务能跑;27B 比较慢,35B-A3B 能到 70 token/s,大多数任务够用。([原文](https://martinfowler.com/articles/exploring-gen-ai/local-models-for-coding-experiences.html))Austin 补充实测经验:对"本地模型 Vibe Coding"不太感冒——之前号称 Qwen 3.5 跟 OSSGPT 120B 相当,他试了免费的 OSSGPT 120B(平时主要用 Opus 4.8),体验"惨不忍睹",猜 Qwen 3.5 大概也差不多;对 Qwen 3.6/3.7 能不能好很多表示怀疑,而且自己的 Spark 硬件可能都带不动。
